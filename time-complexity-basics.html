Time Complexity Basics
Time complexity: How many operations a piece of code performs.

Let’s take a look at time complexity. Understanding time complexity is important because writing your code to run as quickly as possible (this is called optimization) is essential. Code that takes longer to run is prone to loading issues, especially if multiple users are running that code simultaneously.

There are three different measures of time complexity: Worst-case, average-case, and best-case. When you write code, you want to focus on the worst-case time complexity. However, it's helpful to be aware of what average-case and best-case means.

We use certain notations to express a code's time complexity. For worst-case, we use O-notation. This means, O(value). If the time complexity is constant, it's O(1). If the time complexity is linear, it's O(n). N refers to the input size. If we have a list, and that list has n items, that's what n refers to.

For best-case, we use Omega(value), or Ω(value). For average-case, we use Theta(value), or Θ(value). But again, you want to be focused on worst-case time complexity when it comes to code, because we have to think about how to optimize it.

So a code with a time complexity of O(n^2) takes longer to run than a code with a time complexity of O(n), which takes longer than a code with a time complexity of O(1). Imagine doing the same thing once, versus 5 times, versus 25 (5^2) times. That's what O(1) versus O(n) versus O(n^2) means.

But of course, there are lots of different time complexities. There's logarithmic time complexity, which is O(logn). That's a smaller time complexity than O(n). You can read more in depth about specific time complexities on their respective pages.

Here is a ranking of time complexities from fastest to slowest. To show how exactly the values of O work, let's assume n is 8.

1. O(1): Constant time complexity. Input size is irrelevant, because the code always runs same number of times.
2. O(logn): Logarithmic time complexity. When n=8, O(log(n)) = O(log(8)) = 3 operations.
3. O(n): Linear time complexity. When n=8, O(n) = O(8) = 8 operations.
4. O(nlogn): Loglinear time complexity. When n=8, O(nlogn) = O(8log8) = 24 operations.
5. O(n^2): Quadratic time complexity. When n=8, O(n^2) = O(8^2) = 64 operations.
6. O(2^n): Exponential time complexity. When n=8, O(2^n) = O(2^8) = 256 operations.
7. O(n!): Factorial time complexity. When n=8, O(n!) = O(8*7*6*5*4*3*2*1) = 40,320 operations.

There's also space complexity, which is discussed less often. Space complexity refers to the amount of memory space an algorithm occupies. Space complexity is a similar concept to time complexity, but time complexity is more important in writing efficient code.
