Bubble Sort

Let's look at the bubble sort algorithm, which goes through the array, looks at pairs of elements, compares the individuals in the pairs, and swaps them if necessary, until all the elements in the array have been sorted.

The algorithm is named for how the elements 'bubble' to the end through sorting.

Take a look at this pseudocode for bubble sort, which uses 1-based indexing. If you'd like to see the pseudocode with 0-based indexing instead, use the toggle.

                        
    # Assume A is an array of numbers
    n = length(A)
    for i = 1 to n:
        for j = 1 to n-i:
            if A[j] > A[j+1]:
                swap(A[j], A[j+1])
                        
                    
                        
    # Assume A is an array of numbers
    n = length(A)
    for i = 0 to n - 1:
        for j = 0 to n - 1 - i:
            if A[j] > A[j+1]:
                swap(A[j], A[j+1])

The Python implementation of this, with 0-based indexing, would be: 

                    
    # Assume arr is an array of numbers
    n = len(A)
    for i in range(n-1):
        for j in range(n-i-1):
            if A[j] > A[j+1]:
                A[j], A[j+1] = A[j+1], A[j]

Let's return to the pseudocode version, with 1-based indexing. Let's go through the first iteration of the inner loop, with the array of [4, 8, 3].

                    
    for i = 1 to n: # i = 1
        for j = 1 to n - i: # j = 1
            if A[j] > A[j+1]: # A[j] = 4, A[j+1] = 8
                swap(A[j], A[j+1]) # 4 is not greater than 8, so no swap

The array remains unchanged: [4, 8, 3].

Now let's look at the second iteration of the inner loop.

                    
    for i = 1 to n: # i = 1
        for j = 1 to n - i: # j = 2
            if A[j] > A[j+1]: # A[j] = A[2] = 8, A[j+1] = A[3] = 3
                swap(A[j], A[j+1]) # 8 is greater than 3, so swap

Now the array reads: [4, 3, 8].

The inner loop stops one before the end, because there are no more elements that need to be compared. Now we move on to the second iteration of the outer loop, which begins the second set of iterations of the inner loop.

                    
    for i = 1 to n: # i = 2
        for j = 1 to n - i: # j = 1, n - i = 1
            if A[j] > A[j+1]: # A[j] = A[1] = 4, A[j+1] = A[2] = 3
                swap(A[j], A[j+1]) # 4 is greater than 3, so swap

Now the array reads: [3, 4, 8].

The next iteration of the inner loop does not run. This is because the next iteration would be for j = 2 to 1, which is not possible (empty range), so it terminates.

Then the outer loop would begin again, with i = 3. But note where the issue would be:

                    
    for i = 1 to n: # i = 3
        for j = 1 to n - i: # j = 1, n - i = 0

In this final iteration of the outer loop, regardless of j's value, the inner loop does not run. This is because the endpoint of the loop will always be 0, so the start point would be greater than the endpoint, resulting in an empty range. So the final iteration of the outer loop is wasted.

Therefore, the code is functionally equivalent to:

                    
    for i = 1 to n - 1:
        for j = 1 to n - i:
            if A[j] > A[j+1]:
                swap(A[j], A[j+1])

Some pseudocodes include the wasted iteration, while some don't. But seeing it written both ways, you can understand why it stops at one before n -- it's because the last iteration is functionally useless.

Now, let's discuss bubble sort's time complexity.

Bubble sort uses a nested loop. In a worst-case scenario, when the elements are in reverse order, the outer and inner loops run the maximum number of times. There would be a linear time complexity of O(n) for each individual loop, so we multiply O(n) for inner by O(n) for outer.

In the average case, the inner loop and outer loop would, on average, run proportionally Θ(n) times, so we would again multiply Θ(n) for inner by Θ(n) for outer.

In a best-case scenario, all the elements are sorted and we are using an optimized version of the algorithm that uses a flag to terminate the loop when necessary and does not include the wasted outer loop iteration.

Example of optimized version:

                    
    # Assume A is an array of numbers
    n = length(A)
    for i = 1 to n - 1:
        swapped = False
        for j = 1 to n-i:
            if A[j] > A[j+1]:
                swapped = True
                swap(A[j], A[j+1])
        if not swapped:
            break

The outer loop would only need to run once, making its time complexity functionally equivalent to an algorithm with a single loop (linear).

This means that the best-case time complexity is Ω(n), the average-case time complexity is Θ(n²), and the worst-case time complexity is O(n²). The general time complexity for the algorithm is O(n²).
