Constant Time Complexity

Constant time complexity is O(1).

A constant time complexity means each line only runs once. “Constant” refers to how it always runs the same amount of times, opposed to cases of linear time complexity where O(n) is dependent on the size of n.
Let’s look at some examples of constant time complexity.

                    
                food = ["fries", "pizza", "burger"]
                print(food[0])

Line 1 initializes the list food. Simply initializing is always constant time.

Line 2 accesses the first value (value at 0th index) of food and prints it.

This code has an O(1) time complexity, since no lines have to iterate through anything. Accessing an item at an index in an array is always done in constant time, as it’s not doing any looping.

Let’s look at another example.

                    
                flowers = ["rose", "daisy", "sunflower"]
                flowers.append("violet")

Line 1 initializes the list flowers.

Line 2 adds “violet” to the end of the list flowers, so that flowers now would consist of [“rose”, “daisy”, “sunflower”, “violet”].

This code’s time complexity is O(1) because append() is a constant time operation, performing the opposite operation as pop().

Let’s look at an example that can get complex.

                    
                names = ["Anna", "Joe", "Will"]
                names.pop()

Line 1 initializes the list names, which is constant time.

Line 2 removes the item at the last index, which would be “Will.”

This code also has a O(1) time complexity, since pop(), without any argument, is a constant time operation that requires no iterating or repeating.

However, this is only the case when pop() is not passed an argument, as in, when the parenthesis are empty.

In cases where pop() does have an argument that pops the element that is not the last in the list, the time complexity will increase to O(n) to account for having to shift the positions of the list’s other elements. You can read a more thorough explanation of this on the Linear Time Complexity page.

You can access a list of constant operations' time complexities on the official Python website. Additionally, basic arithmetic operations that do not involve exponentials, such as

                    
                x = 2
                result = x * 2
                print(result)

also have a time complexity of O(1), assuming the given numbers are not so large that they exceed a machine's word size.
